批量视频组合拼接工具 用户使用说明

一、软件用途

本软件用于：

1. 导入多个完整视频素材
2. 按固定的 3 层组合规则批量生成多个导出任务
3. 导出一份 DaVinci Resolve 可执行的 Lua 脚本
4. 由用户在 DaVinci Resolve 中手动运行脚本，完成批量导入、创建时间线和渲染

注意：

- 本软件不做视频切割
- 不做字幕
- 不做转场
- 不做 AI 识别
- 不会直接自动控制 Resolve
- 当前版本采用“导出脚本 + 手动在 Resolve 中运行”的方式


二、使用前准备

请先确认：

1. 电脑是 Windows 系统
2. 已安装 DaVinci Resolve
3. 已经会在 DaVinci Resolve 中打开项目
4. 已准备好需要组合的视频素材

推荐导入格式：

- mp4
- mov
- mkv
- avi
- mxf
- mts
- m2ts

说明：

- 如果选择了不在推荐范围内的格式，软件会弹出提示并跳过
- 即使扩展名在推荐范围内，实际是否能被 Resolve 免费版正常导入，也仍取决于视频编码兼容性


三、界面参数说明

1. 素材列表

- 显示当前导入的全部视频素材
- 可以调整顺序
- 这个顺序会影响后续组合结果

2. 每条输出使用素材数 K

- 表示每个导出视频要使用多少个完整素材

3. 导出任务数量

- 表示希望生成多少条导出任务
- 如果填写数量超过当前最大值，软件会自动调整到最大可生成数量

4. 统计区

界面中会显示：

- N：素材总数
- G：完整分组数
- L1：第一层任务数
- L2：第二层任务数
- L3：第三层任务数
- T：总任务数
- 忽略素材：无法组成完整分组而被忽略的素材数量

公式如下：

- G = floor(N / K)
- L1 = G
- L2 = C(G, K) * K^2
- L3 = G * C(G-1, K-2) * C(K,2) * K^(K-2)
- T = L1 + L2 + L3

说明：

- 只有前 G*K 个素材会参与组合
- 剩余素材会被忽略，并显示在“忽略素材”中

5. 视频输出目录

- 表示 Resolve 最终导出视频保存到哪里

6. 导出比例

当前支持：

- 竖屏 9:16
- 竖屏 3:4
- 横屏 16:9
- 横屏 4:3
- 方形 1:1

7. 脚本说明

- 当前脚本固定导出为：
  generated_scripts/batch_concat_resolve_job.lua
- 如果重复导出，会覆盖上一份同名脚本
- 这不会删除已经导出好的视频
- 但会替换这份待执行脚本的内容


四、当前组合规则

当前版本使用 3 层组合逻辑。

第 1 层 L1：

- 按素材导入顺序，每 K 个素材分成一组
- 每组直接生成 1 条任务

第 2 层 L2：

- 从完整分组中选 K 组
- 构造 K x K 矩阵
- 生成 shift=0 到 shift=K-1 的全部稳定任务

第 3 层 L3：

- 恰好有 1 个分组提供 2 个素材
- 剩余 K-2 个素材分别来自不同的其它分组
- 也就是：
  2 + 1 + 1 + ... + 1

导出顺序固定为：

1. 先导出全部 L1
2. 再导出全部 L2
3. 最后导出全部 L3

所以当“导出任务数量”比较小时，会优先取到 L1，然后才进入 L2，再进入 L3。


五、操作步骤

第 1 步：打开软件

运行程序后，会看到主界面。

第 2 步：添加视频

点击“添加视频”，选择要导入的素材。

如果有不支持的扩展名，软件会弹窗提示并自动跳过。

第 3 步：调整素材顺序

使用“上移”“下移”按钮调整顺序。

第 4 步：设置 K

例如：

- K=3：每条输出由 3 个素材组成
- K=4：每条输出由 4 个素材组成

第 5 步：查看统计

确认：

- N
- G
- L1
- L2
- L3
- T
- 忽略素材

第 6 步：设置导出任务数量

如果超出最大值，软件会自动调整。

第 7 步：选择视频输出目录

这个目录用于保存 Resolve 最终导出的视频。

第 8 步：选择导出比例

根据需要选择竖屏、横屏或方形比例。

第 9 步：点击“导出脚本”

导出完成后，界面日志中会显示：

- 脚本路径
- 视频输出目录
- Lua 运行代码
- 每个任务的层级、类型、分组和素材路径


六、在 DaVinci Resolve 中运行脚本

1. 打开 DaVinci Resolve
2. 打开一个项目
3. 打开 Lua Console
4. 粘贴软件日志中给出的 Lua 运行代码
5. 执行

执行成功后，脚本会自动：

- 导入素材
- 创建多个时间线
- 按任务顺序追加素材
- 加入渲染队列
- 开始渲染


七、导出设置说明

当前默认导出格式：

- mp4

脚本会优先尝试：

- mp4 + H.264

如果 Resolve 返回失败，会自动再尝试：

- mp4 + H264


八、关于 CacheClip 和 .gallery

脚本会尝试把 Resolve 产生的以下目录重定向到视频输出目录中：

- CacheClip
- .gallery

说明：

- 这是最佳努力策略
- 是否完全生效，取决于当前 Resolve 版本和项目设置项


九、常见问题

1. 为什么有些素材被忽略？

因为当前规则要求按 K 个素材组成完整分组。
不能组成完整分组的剩余素材会被忽略。

2. 为什么重复导出后脚本内容变了？

因为当前脚本文件名固定。
重复导出时，新的脚本会覆盖上一份脚本。

3. 为什么脚本能导出，但 Resolve 里导入失败？

因为扩展名通过不代表编码一定兼容。
某些 mp4/mov 文件虽然扩展名正确，但编码参数可能不被当前 Resolve 免费版支持。

4. 为什么导出任务数量被自动改小？

因为你填写的数量超过了当前可生成总数 T。
软件会自动调整到最大允许值，避免出错。


十、当前版本说明

当前版本已经实现：

- GUI 导入素材
- 调整顺序
- 设置 K
- 统计 N/G/L1/L2/L3/T
- 提示不支持格式
- 选择输出目录和比例
- 导出 Resolve Lua 脚本

当前版本暂未实现：

- 一键自动连接 Resolve
- 一键自动在 Resolve 内执行脚本


十一、如何启动打包后的程序

如果你拿到的是打包后的 Windows 版本，请进入：

- dist\BatchVideoConcatTool

直接双击：

- BatchVideoConcatTool.exe

说明：

- Python 运行环境已经一起打包在这个目录里
- 普通 Windows 用户不需要单独安装 Python
- 不要只拷贝 exe，一个完整的 dist\BatchVideoConcatTool 文件夹要一起保留
- 用户使用说明.txt 也建议和 exe 放在同一个目录里
